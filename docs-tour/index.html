Foundations 01

Welcome and first orientation

Start by learning the shape of Cursor's agent-first interface. This is the quickest mental map for a new learner: where the chat lives, how the editor and agent surfaces relate, and what Cursor 3 added.

Where to find it: Cmd+Shift+P then search for Open Agents Window.

Foundations 02

How models fit into the UI

You do not need to memorize token pricing on day one, but you should know that model choice changes cost, speed, and depth. The model picker lives right next to the chat workflow, not in a separate product area.

Where to find it: open the chat sidepanel, then use the model picker or the mode menu shortcuts from the keyboard shortcuts page.

Source docs Module

Foundations 03

Your first useful chat

The first useful step is opening the sidepanel and asking a concrete question. This section anchors the habit: chat is a working surface, not just a help popup.

Where to find it: Cmd+I or Cmd+L opens the AI sidepanel.

Source docs Module

Foundations 04

Tab completion

Tab is the fastest Cursor feature to try. Accept, reject, or partially accept suggestions while staying in the editor. It should feel like a code-aware autocomplete, not a chat flow.

Where to find it: type in the editor, then press Tab to accept a suggestion or Esc to reject it.

Source docs Module

Foundations 05

Ask mode

Ask mode is for safe exploration. Use it when you want explanation, architecture help, or codebase understanding without letting Cursor edit anything.

Where to find it: open the sidepanel, then press Shift+Tab until Ask mode is selected.

Source docs Module

Foundations 06

Agent mode and checkpoints

Agent mode is where Cursor can inspect files, edit code, and run verification. The docs clip below also hints at the bigger workflow: one active task, queued follow-ups, and review points.

Where to find it: open the sidepanel, then switch to Agent in the mode picker.

Source docs Module

Core 07

Context and @-mentions

Context is how you steer the model toward the right files, folders, and symbols. For beginners, this is the difference between a vague question and a grounded one.

Where to find it: type @ in chat to reference files, folders, code symbols, docs, or the codebase.

Source docs Module

Core 08

Plan mode

Plan mode is the right move when a task touches many files or the request is still fuzzy. It forces the work to become reviewable before any code is changed.

Where to find it: Shift+Tab rotates to Plan mode in the chat input.

Source docs Module

Core 09

Debug mode

Debug mode changes the posture from "guess a fix" to "gather runtime evidence." Use it for bugs you can reproduce but cannot explain yet.

Where to find it: rotate modes with Shift+Tab until Debug is selected.

Source docs Module

Core 10

Rules

Rules are persistent instructions for the agent. This is the part of Cursor that turns one-off prompts into a reusable project-specific setup.

Where to find it: Cursor Settings, then Rules and Commands.

Source docs Module

Core 11

Skills and slash commands

Skills package repeatable workflows. In Cursor Academy, they are what make onboarding, hints, learning, and progress feel like built-in commands.

Where to find it: type / in chat to invoke skills manually, or view them in Cursor Settings under Rules.

Source docs Module

Core 12

MCP and connected tools

MCP connects Cursor to external tools and data. The most important UI moment for learners is seeing that tool calls are visible, reviewable, and not hidden magic.

Where to find it: tool approval and settings live in chat and Cursor Settings under Model Context Protocol.

Source docs Module

Advanced 13

Subagents and delegation

Subagents let Cursor split noisy work into separate contexts. The UI takeaway is simple: one main conversation can still delegate research, browser, and shell work in parallel.

Where to find it: ask Agent to delegate context-heavy work or create custom subagents in .cursor/agents/.

Source docs Module

Advanced 14

Agents Window and handoff

This is the Cursor 3 hero workflow. Use the Agents Window when you want to manage multiple tasks, move between local and cloud, and stay agent-first instead of file-first.

Where to find it: Cmd+Shift+P, then Open Agents Window.

Source docs Module

Advanced 14

Cloud agents

Cloud agents are what let work keep moving after you leave the editor. They work especially well with the Agents Window, GitHub comments, Slack, and other async workflows.

Where to find it: choose Cloud under the agent input or use cursor.com/agents.

Source docs Module

Advanced 15

Model selection and cost awareness

Model choice matters more as tasks get bigger. Use cheaper defaults for routine work and stronger models when the task really needs more depth or context.

Where to find it: the model picker in the chat input and the usage pages in Cursor settings and dashboard.

Source docs Module

Advanced 16

Hooks and lifecycle automation

Hooks let you attach predictable checks to agent behavior. This is one of the strongest ways to make an AI workflow feel like part of an actual team process instead of a personal prompt habit.

Where to find it: configure hooks in .cursor/hooks.json and review the lifecycle behavior in the docs.

Source docs Module

Advanced 17

Automations and triggered work

Automations use cloud agents on a schedule or on external events like GitHub PRs and Slack messages. They are the natural next step after skills and hooks.

Where to find it: cursor.com/automations or the marketplace automation templates.

Source docs Module

Advanced 18

Custom workflows

This is where everything comes together: rules, skills, plans, hooks, browser work, and cloud handoff. Cursor becomes much easier when you stop treating features as isolated tricks.

Where to find it: combine the modules above, then design your own repeatable workflow inside this repo.
